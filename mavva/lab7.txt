1. Процесс тестирования - это процесс проверки соответствия программного продукта или системы требованиям и ожиданиям пользователей. 

2. TestNG и JUnit - это два фреймворка для тестирования Java-приложений. 

3. Юнит-тестирование - это процесс тестирования отдельных модулей или компонентов программного обеспечения для проверки их работоспособности и соответствия требованиям

4. Тест-кейс - это документ, который описывает шаги, необходимые для выполнения определенного теста в рамках процесса тестирования. 

5. @Test - это аннотация в TestNG и JUnit, которая помечает методы как тестовые и указывает фреймворку, что они должны быть выполнены во время тестирования. 

6. Test suite - это набор связанных тестовых кейсов, которые выполняются вместе как одно целое. 

7. Assert class обеспечивает проверку утверждений во время выполнения теста. Этот класс содержит набор методов, которые можно использовать для проверки, соответствует ли фактическое значение ожидаемому значению.

8. Аннотации @BeforeSuite и @AfterSuite используются для выполнения методов до и после выполнения всех тестов в test suite соответственно. @BeforeTest и @AfterTest используются для выполнения методов до и после запуска тестовых методов, которые относятся к определенному <test> в testng.xml файле.
 
9. Параметризация тестов позволяет запускать один и тот же тестовый метод с различными наборами входных данных. Это позволяет уменьшить объем кода тестирования

10. TDD (Test-Driven Development) - это методология разработки программного обеспечения, которая основана на написании тестов перед написанием кода.
